## Что это за проект

«Сказочник» — лендинг + веб-приложение для генерации персональных сказок для детей.
Ребёнок становится главным героем истории. Есть сказки, загадки, письма от
персонажей, раскраски и обучающие карточки.

## Структура (важно!)

Проект максимально простой — **без сборки, без npm, без фреймворков**:

- `index.html` — вся страница: разметка + CSS (инлайн в `<style>`) + JS (инлайн в `<script>`). ~2360 строк.
- `worker.js` — бэкенд на Cloudflare Worker. Хранит API-ключи как секреты.
- `README.md` — краткое описание.

Всё держим в этих файлах. Не добавляем сборщики, зависимости или новые файлы
без явной просьбы пользователя.

## Фирменный стиль (дизайн-токены)

Цвета объявлены в `:root` внутри `index.html`. Использовать только эти переменные,
не вводить произвольные цвета:

```
--bg:#FFFBF5      тёплый фон страницы
--warm:#FFF3E8    тёплый акцентный фон
--coral:#FF6B6B   главный акцент (кнопки)
--coral-dark:#E85555
--teal:#4ECDC4
--gold:#FFD93D    золото (звёзды, акценты)
--purple:#A78BFA  аудио/загадки
--blue:#60A5FA
--pink:#F9A8D4    письма
--orange:#FB923C
--green:#6BCB77
--text:#2D1B0E    основной текст
--soft:#6B5744    приглушённый текст
--muted:#A89580   подписи
--border:#F0E6D9  границы карточек
```

**Шрифты** (подключены через Google Fonts):
- `Nunito` — основной текст, кнопки, интерфейс.
- `Playfair Display` (serif) — крупные заголовки.
- `Literata` (serif) — текст самих сказок и писем.

**Визуальный язык:**
- Тёплая, уютная, детская, но не "кричащая" эстетика.
- Скруглённые углы у карточек и кнопок (16–28px).
- Мягкие тени: `0 8px 32px rgba(0,0,0,.1)` и подобные.
- Лёгкая анимация при наведении: `transform:translateY(-2px..-6px)`.
- Hero — ночной градиент (тёмно-синий → фиолетовый → коралл) с луной и звёздами.
- Эмодзи как иллюстрации (🌙 ⭐ 🖍️ 🔤 и т.п.).

## Правила работы с кодом

- Новые блоки/секции делать **в том же стиле** и с теми же CSS-переменными.
- CSS добавлять в существующий `<style>`, JS — в существующий `<script>`.
- Имена классов/переменных и техкомментарии — на английском, как в текущем коде
  (классы вида `.product-card`, `.hero-stat-num`).
- Тексты для пользователей на странице — на русском.
- Адаптивность обязательна: проверять, что блок не ломается на мобильном
  (есть медиа-запросы в конце `<style>`).
- Не трогать рабочую логику генерации без причины (см. историю PR #35/#36 про
  разбивку текста на страницы — была откатана).
